Multilingue

Giuliomax Menu Builder prend en charge nativement les sites multilingues. Lorsque WPML, Polylang, TranslatePress ou Multilingual Press est actif, l’extension détecte les langues disponibles et vous permet d’attribuer un libellé distinct à chaque élément de menu, pour chaque langue.

Fonctionnement

Lorsqu’une extension multilingue compatible est active, Menu Builder détecte automatiquement la liste des langues disponibles. Dans la section Multilingue du panneau d’administration, vous pouvez consulter les langues détectées.

Chaque élément de menu dans Menu Builder → Structure du menu dispose d’un champ de libellé distinct pour chaque langue détectée. Le JavaScript du front-office lit la langue active du visiteur et affiche le bon libellé, sans rechargement de page ni menu séparé.

ℹ️

Si aucun libellé n’est défini pour la langue du visiteur, l’extension se rabat sur le libellé par défaut.

Configurer des libellés par langue

  1. Installez et activez une extension multilingue prise en charge (voir la liste ci-dessous).
  2. Ouvrez Menu Builder dans l’administration de WordPress.
  3. Cliquez sur un élément de menu dans la section Structure du menu pour le déployer.
  4. Un champ de libellé apparaît pour chaque langue détectée. Renseignez le libellé traduit pour chaque langue.
  5. Cliquez sur Enregistrer le menu.

Extensions multilingues prises en charge

L’extension détecte automatiquement les langues à partir des extensions suivantes :

  • WPML — détecté via ICL_SITEPRESS_VERSION
  • Polylang — détecté via pll_languages_list()
  • TranslatePress — détecté via la classe TRP_Translate_Press
  • Multilingual Press — détecté via mlp_get_available_languages()

Si aucune extension multilingue n’est active, l’extension utilise la locale par défaut du site.

Langues RTL (arabe, hébreu, persan, etc.)

L’extension intègre une prise en charge complète du RTL (de droite à gauche). Lorsque WordPress détecte que la langue active est en RTL (via la fonction is_rtl()), MenuX procède automatiquement aux opérations suivantes :

  • Inversion de la mise en page du menu (les éléments s’enchaînent de droite à gauche).
  • Inversion de la position des menus déroulants (les panneaux s’ouvrent vers la gauche au lieu de la droite).
  • Inversion du sens du panneau coulissant (il apparaît depuis la droite).
  • Inversion de la position des icônes (icône du panier, icône de recherche).

Aucune configuration n’est nécessaire : le mode RTL s’active automatiquement selon le réglage de langue de WordPress.

Traduire les chaînes d’interface de l’extension

Les chaînes d’interface propres à l’extension (par exemple le libellé « Fermer » du bouton de menu mobile) sont prêtes à être traduites via le système standard de fichiers .po / .mo de WordPress.

Le domaine de traduction (text domain) de l’extension est giuliomax-menu-builder. Pour ajouter une traduction :

  1. Téléchargez le fichier modèle .pot depuis le dossier /languages de l’extension.
  2. Ouvrez-le dans Poedit ou n’importe quel éditeur de fichiers .po.
  3. Enregistrez votre traduction sous le nom giuliomax-menu-builder-{locale}.po (par exemple giuliomax-menu-builder-it_IT.po) et compilez-la en .mo.
  4. Téléversez les deux fichiers dans /wp-content/languages/plugins/.

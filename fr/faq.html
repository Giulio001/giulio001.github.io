Foire aux questions

Réponses aux questions les plus courantes sur Giuliomax Menu Builder.

Généralités

L’extension est-elle vraiment gratuite ? Existe-t-il une version Pro ?

Oui, entièrement gratuite. Il n’existe pas de version Pro, pas de modules complémentaires premium et aucune fonctionnalité bloquée derrière un paywall. Tout ce qui est décrit dans cette documentation est disponible dès le premier jour, sans frais.

Où se trouve le code source de l’extension ?

L’extension est open source. Le dépôt se trouve à l’adresse github.com/Giulio001/menux-free-version. Les rapports de bogues et les pull requests sont les bienvenus.

Fonctionne-t-elle avec n’importe quel thème WordPress ?

Oui. L’extension génère son propre balisage et ses propres styles, indépendamment du thème actif. La seule considération propre au thème est l’endroit où vous placez le menu : pour les thèmes classiques, vous utilisez le shortcode ou un widget ; pour les thèmes FSE, vous utilisez le bloc Gutenberg dans l’éditeur de site (Site Editor).

Va-t-elle ralentir mon site ?

Non. L’extension ne charge que le CSS et le JavaScript nécessaires aux fonctionnalités que vous avez activées. Si vous n’utilisez pas le compte à rebours, son script n’est pas chargé. Les ressources sont minifiées et servies depuis votre serveur (aucun appel CDN vers des services tiers).

Installation et compatibilité

De quelle version de WordPress ai-je besoin ?

WordPress 5.8 ou une version supérieure. Pour profiter de la meilleure expérience d’édition complète du site (FSE) avec le bloc Gutenberg, WordPress 6.0 ou une version supérieure est recommandé.

Fonctionne-t-elle avec les constructeurs de pages (Elementor, Divi, Beaver Builder) ?

Oui, via le shortcode [menux]. Dans Elementor, utilisez le widget Shortcode et collez [menux]. Dans Divi, utilisez le module Code. Dans Beaver Builder, utilisez le module HTML.

Entre-t-elle en conflit avec d’autres extensions de menu ?

Elle peut coexister avec d’autres extensions de menu, tant qu’elles ne sont pas toutes deux appliquées au même emplacement de menu. Si vous utilisiez auparavant une autre extension (par exemple Max Mega Menu ou UberMenu), désactivez-la avant d’activer Giuliomax Menu Builder afin d’éviter les conflits de style.

Puis-je l’utiliser sur une installation multisite ?

Oui. Activez-la à l’échelle du réseau pour la rendre disponible sur tous les sites, ou activez-la site par site. Les réglages sont stockés par site, et non à l’échelle du réseau.

Thèmes et style

Puis-je créer mes propres thèmes ?

Vous pouvez personnaliser en profondeur n’importe quel thème intégré à l’aide des contrôles de style de Menu Builder (sections Couleurs / Typographie / Mise en page). L’enregistrement de ces personnalisations sous forme de « préréglage » nommé (Export → Enregistrer comme préréglage) est prévu pour une future version.

Ma police personnalisée ne figure pas dans la liste des 70+ Google Fonts. Puis-je l’utiliser ?

Oui. Chargez votre police via votre thème, une extension ou en la déclarant manuellement, puis saisissez le nom de la famille de police dans Menu Builder (sections Couleurs / Typographie / Mise en page) → Typographie → Famille de police → Personnalisée. L’extension utilisera la police disponible dans le navigateur.

Comment réinitialiser toutes les modifications de style et revenir aux valeurs par défaut du thème ?

Dans Menu Builder (sections Couleurs / Typographie / Mise en page), cliquez sur Réinitialiser aux valeurs par défaut du thème, en bas du panneau. Cela restaure toutes les valeurs telles que définies par le thème actuellement appliqué.

Méga-menu

Le panneau du méga-menu est coupé sur le côté droit de l’écran. Comment corriger cela ?

Réglez la largeur du panneau sur Pleine largeur dans Menu Builder → Méga-menu → Largeur du panneau. Si vous avez besoin d’un panneau plus étroit, vérifiez que le CSS du conteneur de votre site n’applique pas overflow: hidden à un élément ancêtre du menu.

Puis-je placer un formulaire de contact ou un produit WooCommerce dans une colonne de méga-menu ?

Oui — utilisez le type de contenu de colonne Shortcode et collez n’importe quel shortcode ou code HTML. Par exemple : [contact-form-7 id="123"] ou un shortcode de produit WooCommerce.

Mobile

Le menu de bureau et le menu mobile sont visibles en même temps. Comment corriger cela ?

Cela signifie généralement qu’une autre règle CSS prend le dessus sur les styles de point de rupture de l’extension. Ouvrez les outils de développement du navigateur, inspectez le conteneur du menu et recherchez une propriété display provenant de votre thème. Ajoutez une règle CSS plus spécifique à la feuille de style de votre thème, ou ajustez la valeur du point de rupture dans Menu Builder → Mobile.

WooCommerce

Le compteur du panier ne se met pas à jour sans recharger la page.

Assurez-vous que votre extension de cache (le cas échéant) ne met pas en cache les requêtes AJAX de WooCommerce. La plupart des extensions de cache proposent un mode compatible WooCommerce ou excluent automatiquement les requêtes de fragments de panier — consultez la documentation de votre extension de cache.

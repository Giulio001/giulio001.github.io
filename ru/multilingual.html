Мультиязычность

В Giuliomax Menu Builder встроена поддержка мультиязычных сайтов. Когда активны WPML, Polylang, TranslatePress или Multilingual Press, плагин определяет доступные языки и позволяет задать для каждого пункта меню отдельное название на каждом языке.

Как это работает

Когда активен совместимый мультиязычный плагин, Menu Builder автоматически определяет список доступных языков. В разделе Multilingual панели администратора вы видите обнаруженные языки.

Для каждого пункта меню в разделе Menu Builder → Menu Structure появляется отдельное поле названия для каждого обнаруженного языка. JavaScript на фронтенде считывает активный язык посетителя и отображает нужное название — без перезагрузки страницы и без отдельного меню.

ℹ️

Если название для языка посетителя не задано, плагин использует название по умолчанию.

Настройка названий по языкам

  1. Установите и активируйте один из поддерживаемых мультиязычных плагинов (см. список ниже).
  2. Откройте Menu Builder в админке WordPress.
  3. Нажмите на любой пункт меню в разделе Menu Structure, чтобы развернуть его.
  4. Вы увидите поле ввода названия для каждого обнаруженного языка. Заполните переведённое название для каждого языка.
  5. Нажмите Сохранить меню.

Поддерживаемые мультиязычные плагины

Плагин автоматически определяет языки из следующих плагинов:

  • WPML — определяется через ICL_SITEPRESS_VERSION
  • Polylang — определяется через pll_languages_list()
  • TranslatePress — определяется через класс TRP_Translate_Press
  • Multilingual Press — определяется через mlp_get_available_languages()

Если ни один мультиязычный плагин не активен, плагин использует локаль сайта по умолчанию.

Языки RTL (арабский, иврит, персидский и др.)

Плагин включает полную поддержку RTL (письмо справа налево). Когда WordPress определяет, что активный язык относится к RTL (через функцию is_rtl()), MenuX автоматически:

  • Зеркалирует макет меню (элементы располагаются справа налево).
  • Переворачивает положение выпадающих меню (панели открываются влево, а не вправо).
  • Меняет направление выдвижного меню (выезжает справа).
  • Меняет местами расположение значков (значок корзины, значок поиска).

Никакой настройки не требуется — режим RTL активируется автоматически в зависимости от языковой настройки WordPress.

Перевод строк интерфейса плагина

Собственные строки интерфейса плагина (например, метка «Закрыть» для кнопки мобильного меню) готовы к переводу через стандартную систему файлов .po / .mo в WordPress.

Текстовый домен плагина — giuliomax-menu-builder. Чтобы добавить перевод:

  1. Скачайте файл-шаблон .pot из папки /languages плагина.
  2. Откройте его в Poedit или любом редакторе .po.
  3. Сохраните перевод как giuliomax-menu-builder-{locale}.po (например, giuliomax-menu-builder-it_IT.po) и скомпилируйте в .mo.
  4. Загрузите оба файла в /wp-content/languages/plugins/.

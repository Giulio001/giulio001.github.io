Поддержка WCAG 2.2 AA

Плагин спроектирован вокруг следующих критериев успеха WCAG 2.2, относящихся к навигационным компонентам:

КритерийУровеньКак выполняется
1.4.3 Контрастность (минимальная)AAВстроенные темы спроектированы с расчётом на коэффициент контрастности 4.5:1; собственные переопределения цветов нужно перепроверять.
2.1.1 КлавиатураAВсе функции доступны только с клавиатуры.
2.1.2 Отсутствие клавиатурной ловушкиAФокус всегда можно вывести из меню без использования указывающего устройства.
2.4.3 Порядок фокусаAПорядок фокуса соответствует визуальному порядку чтения меню.
2.4.7 Видимость фокусаAAНа каждом интерактивном элементе отображается видимый индикатор фокуса.
2.4.11 Внешний вид фокусаAAИндикаторы фокуса соответствуют требованиям к минимальному размеру и контрастности (новый критерий в WCAG 2.2).
4.1.2 Имя, роль, значениеAУ всех интерактивных элементов есть доступное имя и корректная роль ARIA.

Атрибуты ARIA

Плагин автоматически добавляет в формируемую разметку следующие атрибуты ARIA:

  • role="navigation" и aria-label для элемента <nav> (метка совпадает с названием меню в WordPress).
  • aria-haspopup="true" и aria-expanded для элементов верхнего уровня, у которых есть выпадающее меню или мегаменю.
  • aria-controls, связывающий кнопку-триггер с её панелью.
  • role="menu" и role="menuitem" внутри выпадающих меню.
  • aria-current="page" для элемента, соответствующего текущей странице.

Ничего из этого настраивать не нужно — атрибуты применяются автоматически на основе структуры меню.

Управление фокусом

  • При открытии выпадающего меню фокус перемещается на первый элемент внутри него.
  • При закрытии выпадающего меню (по Escape или при переводе фокуса в сторону) фокус возвращается на элемент, открывший его.
  • Оверлей мобильного меню удерживает фокус внутри себя, пока открыт (фокус нельзя вывести за пределы панели клавишей Tab). При закрытии оверлея фокус возвращается на кнопку-гамбургер.

Уменьшение анимации

Все переходы и анимации учитывают CSS-медиазапрос prefers-reduced-motion: reduce. Когда пользователь включил режим «Уменьшить движение» в настройках доступности своей операционной системы:

  • Анимации появления/затухания выпадающих меню заменяются мгновенным показом/скрытием.
  • Переходы открытия/закрытия мобильного меню становятся мгновенными.
  • Анимация появления панели объявлений отключается.
  • Анимации появления пунктов меню отключаются.

Проверка вашего меню

После настройки меню используйте следующие инструменты, чтобы проверить доступность:

  • Только клавиатура — отключите мышь и пройдитесь по всему меню, используя только Tab, клавиши со стрелками, Enter и Escape.
  • axe DevTools — бесплатное расширение браузера, которое запускает автоматические проверки WCAG на живой странице.
  • NVDA / VoiceOver — программы чтения с экрана для проверки того, что состояния выпадающих меню и метки озвучиваются корректно.
  • Color Contrast Analyzer — убедитесь, что ваши собственные настройки цветов по-прежнему соответствуют минимальному коэффициенту 4.5:1.

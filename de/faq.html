Häufig gestellte Fragen

Antworten auf die häufigsten Fragen zum Giuliomax Menu Builder.

Allgemein

Ist das Plugin wirklich kostenlos? Gibt es eine Pro-Version?

Ja, vollständig kostenlos. Es gibt keine Pro-Version, keine Premium-Add-ons und keine Funktionen hinter einer Bezahlschranke. Alles, was in dieser Dokumentation beschrieben wird, ist von Anfang an kostenlos verfügbar.

Wo befindet sich der Quellcode des Plugins?

Das Plugin ist Open Source. Das Repository befindet sich unter github.com/Giulio001/menux-free-version. Bug-Reports und Pull Requests sind willkommen.

Funktioniert es mit jedem WordPress-Theme?

Ja. Das Plugin rendert sein eigenes Markup und seine eigenen Stile unabhängig vom aktiven Theme. Die einzige themespezifische Überlegung betrifft die Platzierung des Menüs — für klassische Themes verwendest du den Shortcode oder ein Widget; für FSE-Themes den Gutenberg-Block im Site-Editor.

Verlangsamt es meine Website?

Nein. Das Plugin bindet nur das CSS und JavaScript ein, das für die von dir aktivierten Funktionen benötigt wird. Wenn du den Countdown-Timer nicht verwendest, wird dieses Skript nicht geladen. Assets sind minimiert und werden von deinem Server bereitgestellt (keine CDN-Aufrufe an Drittanbieter-Dienste).

Installation & Kompatibilität

Welche WordPress-Version benötige ich?

WordPress 5.8 oder höher. Für die beste Full-Site-Editing-Erfahrung (FSE) mit dem Gutenberg-Block wird WordPress 6.0 oder höher empfohlen.

Funktioniert es mit Page-Buildern (Elementor, Divi, Beaver Builder)?

Ja, über den [menux]-Shortcode. In Elementor verwende das Widget Shortcode und füge [menux] ein. In Divi verwende das Code-Modul. In Beaver Builder das HTML-Modul.

Verursacht es Konflikte mit anderen Menü-Plugins?

Es kann neben anderen Menü-Plugins koexistieren, solange beide nicht auf dieselbe Menü-Location angewendet werden. Wenn du zuvor ein anderes Plugin verwendet hast (z. B. Max Mega Menu, UberMenu), deaktiviere es vor der Aktivierung von Giuliomax Menu Builder, um Stil-Konflikte zu vermeiden.

Kann ich es bei einer Multisite-Installation verwenden?

Ja. Aktiviere es netzwerkweit, um es auf allen Sites verfügbar zu machen, oder aktiviere es pro Site. Einstellungen werden pro Site gespeichert, nicht netzwerkweit.

Themes & Gestaltung

Kann ich eigene Themes erstellen?

Du kannst jedes eingebaute Theme mithilfe der Stilsteuerungen in Menu Builder (Farben / Typografie / Layout) stark anpassen. Das Speichern dieser Anpassungen als benanntes „Preset" (Export → Als Preset speichern) ist für eine zukünftige Version geplant.

Meine benutzerdefinierte Schriftart steht nicht in der Liste mit 70+ Google Fonts. Kann ich sie trotzdem verwenden?

Ja. Lade deine Schriftart über dein Theme, ein Plugin oder manuell ein, und gib dann den Schriftfamiliennamen unter Menu Builder → Typografie → Schriftfamilie → Benutzerdefiniert ein. Das Plugin verwendet die Schriftart, die im Browser verfügbar ist.

Wie setze ich alle Stiländerungen zurück und kehre zu den Theme-Standardwerten zurück?

Klicke in Menu Builder (Farben / Typografie / Layout) auf Auf Theme-Standardwerte zurücksetzen am unteren Ende des Panels. Dadurch werden alle Werte auf die des aktuell angewendeten Themes zurückgesetzt.

Mega-Menü

Das Mega-Menü-Panel wird am rechten Bildschirmrand abgeschnitten. Wie behebe ich das?

Setze die Panel-Breite auf Volle Breite unter Menu Builder → Mega-Menü → Panel-Breite. Wenn du ein schmaleres Panel benötigst, überprüfe, ob das Container-CSS deiner Website overflow: hidden auf ein übergeordnetes Element des Menüs anwendet.

Kann ich ein Kontaktformular oder ein WooCommerce-Produkt in eine Mega-Menü-Spalte einfügen?

Ja — verwende den Spalteninhalt-Typ Shortcode und füge einen beliebigen Shortcode oder HTML ein. Zum Beispiel: [contact-form-7 id="123"] oder einen WooCommerce-Produkt-Shortcode.

Mobilgeräte

Desktop- und Mobilmenü sind gleichzeitig sichtbar. Wie behebe ich das?

Das bedeutet normalerweise, dass eine andere CSS-Regel die Breakpoint-Stile des Plugins überschreibt. Öffne die Browser-DevTools, untersuche den Menü-Container und suche nach einer display-Eigenschaft aus deinem Theme. Füge eine spezifischere CSS-Regel zu deinem Theme-Stylesheet hinzu oder passe den Breakpoint-Wert unter Menu Builder → Mobile an.

WooCommerce

Die Warenkorbanzahl wird ohne Neuladen der Seite nicht aktualisiert.

Stelle sicher, dass dein Caching-Plugin (falls vorhanden) keine WooCommerce-AJAX-Anfragen zwischenspeichert. Die meisten Caching-Plugins haben einen WooCommerce-kompatiblen Modus oder schließen Warenkorb-Fragment-Anfragen automatisch aus — lies die Dokumentation deines Caching-Plugins.

Veelgestelde vragen

Hieronder vind je antwoorden op de meestgestelde vragen over Giuliomax Menu Builder. Staat jouw vraag er niet bij? Neem contact op via het supportformulier in het WordPress-dashboard.

Compatibiliteit

Werkt de plugin met elk WordPress-thema?

Giuliomax Menu Builder vervangt het standaard navigatie-element van je thema. De plugin werkt met de meeste thema's, maar sommige thema's laden hun eigen navigatiestijlen die kunnen interfereren. Voeg in dat geval de CSS-klasse .menux-nav toe aan de CSS-specificiteitsregels van je thema om conflicten op te lossen.

Is de plugin compatibel met Gutenberg en de Site Editor?

Ja. Je kunt het Giuliomax Menu Builder-blok toevoegen via de blokken-invoeginterface in zowel de klassieke editor, de post-editor als de Full Site Editor. In de Site Editor gebruik je het blok in een header-sjabloongedeelte.

Werkt het met WooCommerce?

Ja, de winkelwagen-widget integreert native met WooCommerce. Je hebt WooCommerce 6.0 of hoger nodig. Het aantal artikelen en het subtotaal worden live bijgewerkt via AJAX zonder paginaherlading.

Welke WordPress-versie is vereist?

Giuliomax Menu Builder vereist WordPress 6.0 of hoger en PHP 7.4 of hoger. We raden aan altijd de nieuwste stabiele versie van WordPress te gebruiken.

Installatieproblemen

De shortcode geeft niets weer op de pagina.

Controleer of je het juiste menu-ID hebt opgegeven in de shortcode ([menux id="123"]). Ga naar Weergave › Menu's om het ID te vinden in de URL wanneer je het menu bewerkt. Zorg ook dat de menulocatie correct is gekoppeld.

Het menu wordt twee keer weergegeven.

Dit gebeurt als zowel je thema als de Giuliomax Menu Builder-shortcode of het blok hetzelfde menu weergeven. Schakel de standaard navigatie van je thema uit via Weergave › Aanpassen › Menu's of verwijder de wp_nav_menu()-aanroep in de header.php van je thema.

Stijlen worden niet geladen na activatie.

Ledig de cache van je cachingplugin (bijv. WP Super Cache, W3 Total Cache) en eventuele CDN-caches. Als je een minificatieplugin gebruikt, voeg dan de CSS- en JS-bestanden van Giuliomax toe aan de uitsluitingslijst om conflicten te voorkomen.

Uiterlijk en stijl

Hoe pas ik de kleur van het menu aan?

Ga naar Giuliomax › Uiterlijk in het dashboard. Daar vind je kleurkiezers voor achtergrond, tekst, hover-kleur, accent en meer. Alle wijzigingen worden opgeslagen als CSS-variabelen en zijn direct zichtbaar op de frontend.

Kan ik aangepaste CSS toevoegen?

Ja. Gebruik het veld Aangepaste CSS in de plugin-instellingen of het WordPress-aanpassingsscherm (Weergave › Aanpassen › Aanvullende CSS). Target het menu via .menux-nav of specifiekere selectors zoals .menux-nav .menux-item.

De lettertypen komen niet overeen met mijn thema.

De plugin erft standaard het lettertype van je thema via font-family: inherit. Als dit niet het geval is, controleer dan of een CSS-resetbestand dit overschrijft. Voeg .menux-nav { font-family: var(--jouw-thema-font, inherit); } toe aan je aangepaste CSS.

Prestaties

Vertraagt de plugin mijn website?

Giuliomax Menu Builder is geoptimaliseerd voor prestaties. CSS en JS worden minified aangeleverd en alleen geladen op pagina's waar het menu actief is. De scriptlading is uitgesteld (defer) zodat de kritieke renderingweg niet wordt geblokkeerd.

Kan ik de plugin-assets samenvoegen met andere scripts?

Ja, via je cachingplugin of een bundeltool. Zorg er wel voor dat de laadvolgorde correct is: de CSS van Giuliomax moet na het reset-stylesheet van je thema komen, en het JS-bestand na jquery (indien van toepassing).

Licentie en updates

Hoe ontvang ik updates?

Updates worden automatisch aangeboden via het WordPress-updatessysteem zodra een nieuwe versie beschikbaar is. Ga naar Dashboard › Updates om handmatig te controleren op beschikbare updates.

Kan ik de plugin op meerdere websites gebruiken?

Raadpleeg de licentievoorwaarden die bij je aankoop zijn meegeleverd. De gratis versie mag op onbeperkte websites worden gebruikt. Voor premium licenties gelden specifieke domeinbeperkingen zoals beschreven in je licentiedocument.

Wat gebeurt er met mijn instellingen als ik de plugin deactiveer?

Deactiveren verwijdert geen instellingen. Alle configuraties blijven bewaard in de WordPress-database. Verwijderen (via Plugins › Geïnstalleerde plugins › Verwijderen) verwijdert standaard alle plugin-gegevens. Als je de gegevens wilt bewaren, exporteer ze dan eerst via Giuliomax › Importeren / Exporteren.
